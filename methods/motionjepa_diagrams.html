Figure · MotionJEPA training objective. The four corners of the diamond carry, top-down and left-right, the current frame It, the future frame It+Δ, the trainable MotionEncoder fθ, and the training-only decoder MotionDiT gφ. Every solid green edge silently passes its frame through the same frozen SigLIP backbone, so each carries a (B, 256, 1152) patch-token grid — drawing the SigLIP module is omitted to keep the data flow visible. fθ consumes the SigLIP features of both frames and the proprio delta Δst, and emits the motion code mt. gφ takes the SigLIP features of It, the motion code mt, and a diffusion timestep k, and is supervised against the SigLIP features of It+Δ via the L2 loss ℒpred. At deployment gφ is discarded — only mt is exposed to the policy.
