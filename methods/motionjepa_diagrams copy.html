FrozenSigLIP  internals

A fixed SiglipVisionModel wrapped so that no gradient flows. It only exists to turn each frame into a patch-token grid of shape (B, 256, 1152).
input image  (B, 3, H, W)
Forward pass
00 Raw image tensor (B, 3, H, W)
01 Resize to 224 × 224  bilinear (B, 3, 224, 224)
02 Patch embedding  ·  Conv2d  k=14, s=14  ·  flatten to sequence (B, 256, 1152)
03 Add learned positional embedding (B, 256, 1152)
04 N × Transformer encoder blocks  LayerNorm + MHA + MLP (B, 256, 1152)
05 Final LayerNorm (B, 256, 1152)
✓ patch tokens  —  state representation (B, 256, 1152)

Why frozen?

  • All parameters have requires_grad = False
  • Runs under torch.no_grad() at training time
  • Acts as a fixed perceptual tokenizer — the model only learns how to manipulate these tokens, not how to produce them

Key numbers

  • Image side  ·  224
  • Patch size  ·  14 × 14
  • Grid  ·  16 × 16 = 256
  • Embed dim  ·  1152

Used twice

  • Same weights produce current_tokens and future_tokens
  • Downstream trainable modules only ever see these two token tensors — never the raw pixels

Why learned queries?

  • Causal attention lets the M query tokens at the tail attend to every delta token before them
  • They function as read-out heads — each one summarises a different aspect of the current→future change
  • Everything before them is never required downstream, so the model is free to use them as a pure bottleneck

Residual prediction

  • The decoder never outputs an absolute future — it outputs a correction to the current token
  • α0 = 0 ⟹ r = 0 ⟹ x̂ = z = LN(x0)  — the decoder starts as a normalized identity copy and learns corrections
  • Tied input / output projections cut parameters and stabilise learning in the token space

Motion injection modes

  • cross — motion tokens act as K/V in cross-attention; every patch attends to the full motion code
  • film — pool M tokens to a single vector, then channel-wise shift + scale
  • cross + film — both injections in the same block
