\subsection{Model Architecture}
\label{sec:arch}

Our policy is a memory-augmented vision-language-action (VLA) model built on
top of the $\pi_{0.5}$ backbone. We initialize from the pre-trained backbone
and fine-tune its experts end-to-end in Stage~2 (Sec.~\ref{sec:training}).
The only components kept frozen throughout both stages are the SigLIP2 visual
encoder and, after Stage~1, the MotionEncoder $f_\theta$.
We inject the per-episode memory sequence assembled in
Sec.~\ref{sec:memory} as additional context tokens.

\paragraph{$\pi_{0.5}$ backbone.}
At history timestep $t$, the policy receives language tokens
$\boldsymbol{\ell} = \mathrm{Tok}_{\mathrm{text}}(l) \in \mathbb{R}^{N_\ell\times d}$
and multi-view image tokens
$\mathbf{o}_t = \mathrm{Tok}_{\mathrm{img}}(I_t) \in \mathbb{R}^{N_o\times d}$
extracted by a frozen SigLIP2 encoder. $\pi_{0.5}$
follows the $\pi_0$ multi-expert design and is composed
of (i) a \emph{VLM expert} $E_{\mathrm{vlm}}$ that fuses language and vision
and (ii) an \emph{action expert} $E_{\mathrm{act}}$ that predicts an action
chunk conditioned on the VLM features and a denoising timestep $\tau$. Both
experts are $L$-layer transformers connected by a \emph{blockwise causal
attention} operator
\begin{equation}
  \mathbf{y} \;=\; \mathrm{BlockAttn}(\mathbf{x}_1, \mathbf{x}_2, \dots, \mathbf{x}_G),
  \label{eq:blockattn}
\end{equation}
in which attention is bidirectional within each block $\mathbf{x}_i$ and
causal across blocks (block $i$ may attend blocks $1\!:\!i$ but not future
blocks). The action expert is conditioned on $\tau$ via AdaLN-Zero
modulation: with action latent
$\mathbf{s}_t^k$ at layer $k$,
\begin{equation}
  \tilde{\mathbf{s}}_t^k = \mathbf{s}_t^{k-1} + g_\tau \odot
  \mathrm{BlockAttn}^k\!\big(\mathrm{Norm}^k_{\mathrm{vlm}}(\mathbf{u}_t^{k-1}),\,
  \mathrm{Norm}^k_{\mathrm{act}}(\mathbf{s}_t^{k-1},\tau)\big),
  \quad
  \mathbf{s}_t^k = \tilde{\mathbf{s}}_t^k + \tilde{g}_\tau \odot
  \mathrm{MLP}^k_{\mathrm{act}}\!\big(\mathrm{Norm}^k_{\mathrm{act}}(\tilde{\mathbf{s}}_t^k,\tau)\big),
  \label{eq:actexpert}
\end{equation}
where $g_\tau,\tilde g_\tau$ are gates conditioned on $\tau$. Unlike the
original $\pi_{0.5}$, which uses the VLM expert to also generate low-level
language commands, we keep the VLM as a pure feature extractor; this leaves
a clean interface at which memory can be injected.

\paragraph{Memory token block.}
To enable history-dependent reasoning, we augment the policy with an explicit
memory state $\mathbf{M}_t$:
\begin{equation}
  a_t \;\sim\; \pi\!\left(\cdot \;\middle|\; l,\; \mathbf{o}_t,\; \mathbf{M}_t\right),
  \label{eq:mem-policy}
\end{equation}
where $\mathbf{M}_t$ is the length-$B$ projection of the causal prefix
$\mathcal{M}_t$ (Eq.~\ref{eq:mem-causal}), containing only observations and
motion windows that have fully elapsed before step $t$.
For fair comparison across designs, we allocate a fixed memory token budget $B$
for all memory-based models: each representation produces at most $B$ tokens;
if fewer are available, the sequence is zero-padded.
Thus all memory variants share the same interface
\begin{equation}
  \mathbf{M}_t \;\in\; \mathbb{R}^{B\times d}.
  \label{eq:mem-interface}
\end{equation}
We expose the memory sequence $\mathcal{M}$ produced in
Sec.~\ref{sec:memory} to the backbone as this token block.
The two-token-kind structure of $\mathcal{M}$ — TokenDropped keyframe patches
and MotionEncoder motion tokens — is opaque to the backbone: from the policy's
perspective $\mathbf{M}_t$ is just a length-$B$ sequence of $d$-dimensional
vectors carrying time tags.

\paragraph{Three integration mechanisms.}
We systematically study three places at which $\mathbf{M}_t$ can enter the
backbone, trading parameter count, attention budget, and isolation of the
pretrained VLM features against each other.

\emph{(1) Memory-as-Context (MaC).} The simplest variant: concatenate
$\mathbf{M}_t$ with the standard inputs of the VLM expert,
\begin{equation}
  \mathbf{u}_t^0 \;=\; [\,\mathbf{M}_t;\, \mathbf{o}_t;\, \boldsymbol{\ell}\,],
  \qquad
  \mathbf{u}_t \;=\; E_{\mathrm{vlm}}(\mathbf{u}_t^0).
  \label{eq:mac}
\end{equation}
MaC introduces \emph{no} new parameters and reuses the existing blockwise
causal attention (\ref{eq:blockattn}); its cost is that memory tokens
compete with raw observation tokens for the same attention budget inside
$E_{\mathrm{vlm}}$, and any noise in $\mathbf{M}_t$ propagates through every
downstream feature.

\emph{(2) Memory-as-Modulator (MaM).} Instead of feeding memory through the
VLM, we use it to \emph{modulate} the action expert. At each layer $k$, the
action latent $\tilde{\mathbf{s}}_t^k$ first cross-attends $\mathbf{M}_t$,
and the result is mapped to per-feature scale/shift parameters via a
zero-initialized MLP that is then composed with AdaLN:
\begin{align}
  \mathbf{r}_t^k &= \mathrm{Attn}^k_{\mathrm{mod}}\!\big(Q\!=\!\tilde{\mathbf{s}}_t^k,\; K\!=\!\mathbf{M}_t,\; V\!=\!\mathbf{M}_t\big), \label{eq:mam-attn}\\
  (\gamma_t^k,\beta_t^k) &= \mathrm{MLP}^k_{\mathrm{mod}}(\mathbf{r}_t^k), \label{eq:mam-mlp}\\
  \widehat{\mathbf{s}}_t^k &= \gamma_t^k \odot \mathrm{Norm}^k_{\mathrm{mod}}(\tilde{\mathbf{s}}_t^k) + \beta_t^k, \label{eq:mam-adain}\\
  \mathbf{s}_t^k &= \widehat{\mathbf{s}}_t^k + \tilde g_\tau \odot \mathrm{MLP}^k_{\mathrm{act}}\!\big(\mathrm{Norm}^k_{\mathrm{act}}(\widehat{\mathbf{s}}_t^k,\tau)\big). \label{eq:mam-out}
\end{align}
$\mathrm{MLP}^k_{\mathrm{mod}}$ is initialized so that
$(\gamma_t^k,\beta_t^k)\!=\!(1,0)$ at the start of fine-tuning, making the
modulation an exact identity at step zero and protecting the pretrained
$\pi_{0.5}$ behavior. MaM adds a moderate number of parameters
($\sim\!80$M, see Sec.~\ref{sec:impl}) and, importantly, leaves the VLM
expert untouched.

\emph{(3) Memory-as-Expert (MaE).} We allocate a dedicated third expert
$E_{\mathrm{mem}}$ that processes $\mathbf{M}_t$ through its own $L$ layers
of self-attention plus MLP:
\begin{equation}
  \tilde{\mathbf{v}}_t^k = \mathbf{v}_t^{k-1} + \mathrm{BlockAttn}^k\!\big(\mathrm{Norm}^k_{\mathrm{mem}}(\mathbf{v}_t^{k-1})\big),
  \quad
  \mathbf{v}_t^k = \tilde{\mathbf{v}}_t^k + \mathrm{MLP}^k_{\mathrm{mem}}\!\big(\mathrm{Norm}^k_{\mathrm{mem}}(\tilde{\mathbf{v}}_t^k)\big),
  \label{eq:mae-self}
\end{equation}
with $\mathbf{v}_t^0 = \mathbf{M}_t$. The action expert's update
(\ref{eq:actexpert}) is then extended to attend \emph{both} VLM and memory
features as additional context blocks:
\begin{equation}
  \tilde{\mathbf{s}}_t^k = \mathbf{s}_t^{k-1} + g_\tau \odot
  \mathrm{BlockAttn}^k\!\big(
  \mathrm{Norm}^k_{\mathrm{mem}}(\mathbf{v}_t^{k-1}),\,
  \mathrm{Norm}^k_{\mathrm{vlm}}(\mathbf{u}_t^{k-1}),\,
  \mathrm{Norm}^k_{\mathrm{act}}(\mathbf{s}_t^{k-1},\tau)\big).
  \label{eq:mae-cross}
\end{equation}
The blockwise causal mask is configured asymmetrically: the VLM expert and
memory expert each use self-attention only and \emph{do not} attend each
other; the action expert is the sole consumer that attends both. This
asymmetry preserves the pretrained $\pi_{0.5}$ representations exactly while
allocating a separate transformer's worth of capacity ($\sim\!190$M
parameters) to memory processing.

\subsection{Memory Token Generation and Training}
\label{sec:memory}
% Scope: this subsection describes how the per-episode memory tokens consumed
% by the downstream VLA are produced and trained. The downstream injection
% mechanism (how the VLA attends to or interleaves these tokens) is described
% in Sec.~\ref{sec:arch}.

As the robot executes, the memory $\mathcal{M}_\tau$ grows online from the
history of observations accumulated prior to policy step $\tau$.  It is a
sparse, time-aligned sequence of two complementary token kinds:
(i) \emph{keyframe tokens}, which anchor the scene at semantic transitions,
and (ii) \emph{motion tokens}, fixed-stride summaries of short-range
visual change produced by a learned MotionEncoder. Both kinds are encoded
strictly from already-elapsed frames: keyframe tokens are committed as soon
as a keyframe is detected, and a motion token $m_t$ over window
$[t,\,t\!+\!\Delta]$ becomes available only after step $t\!+\!\Delta$ has
passed (Eq.~\ref{eq:mem-causal}). All encoding is performed by frozen
perception backbones independently of the VLA; no gradient flows back into
these encoders during VLA training.

\paragraph{Keyframe detection.}
Let $\{I_t\}_{t \geq 0}\!\subset\!\mathbb{R}^{3\times H\times W}$ denote the sequence of RGB frames observed during execution.
Keyframes mark moments of meaningful scene change (object pickup,
contact events, viewpoint shifts). Each frame is scored by the smoothed
discrete derivative of a frozen CLIP encoder $\Phi_{\mathrm{CLIP}}$:
\begin{equation}
  d_t \;=\; \big\| \Phi_{\mathrm{CLIP}}(I_t) - \Phi_{\mathrm{CLIP}}(I_{t-1}) \big\|_2,
  \qquad
  \tilde d_t \;=\; (k_{\mathrm{rbf}} * d)_t,
\end{equation}
where $k_{\mathrm{rbf}}$ is a small RBF kernel that suppresses frame-rate
jitter. Keyframes are local maxima of $\tilde d_t$ exceeding a threshold
$\tau$, with the first frame $t=0$ always retained as an anchor. The
detector is purely deterministic preprocessing and has no learnable
parameters; $\tau$ is a single hyperparameter chosen so that the average
inter-keyframe gap is $\sim\!32$ frames.

\paragraph{Keyframe TokenDrop.}
Storing the full $S$-token grid for every keyframe is wasteful: most of
the image (background, robot base, table) repeats across keyframes. For
every keyframe $k_i$ with $i\!>\!0$ we retain only the patches that
actually changed relative to its \emph{previous} keyframe $k_{i-1}$.
Concretely, on the $16\!\times\!16$ patch grid we mark a patch as
\emph{significant} if its mean per-pixel absolute difference between
$I_{k_i}$ and $I_{k_{i-1}}$ exceeds $\delta$, and we keep the union of
all significant patches together with the patches lying inside their
axis-aligned bounding rectangle (the latter preserves spatial contiguity
for downstream attention). The first keyframe $k_0$ is always kept in
full ($S$ tokens); each subsequent keyframe contributes
$|\mathcal{P}_i|\!\le\!S$ tokens. TokenDrop is also parameter-free.

\paragraph{Motion tokens.}
Let $\Phi_{\mathrm{SigLIP}}\!:\!\mathbb{R}^{3\times H\times W}\!\to\!\mathbb{R}^{S\times E}$ ($S\!=\!256$ patches, $E\!=\!1152$) be the frozen SigLIP tokenizer, and let $x_t \!=\! \Phi_{\mathrm{SigLIP}}(I_t)\in\mathbb{R}^{S\times E}$; $\{s_t\}$ denotes the proprioceptive state sequence.
Independently of keyframe placement, we tile the video with non-overlapping
windows of fixed stride $\Delta$ (we use $\Delta\!=\!8$). For each window
$[t,\, t\!+\!\Delta]$ a trainable \emph{MotionEncoder}
$f_\theta$ compresses the visual change into $M$ latent tokens
$m_{t}\!\in\!\mathbb{R}^{M\times D_m}$:
\begin{equation}
  m_t \;=\; f_\theta\!\left(\, x_t,\; x_{t+\Delta},\; \Delta s_t \,\right),
  \qquad \Delta s_t = s_{t+\Delta} - s_t.
\end{equation}
Internally, $f_\theta$ (i) computes the patchwise visual delta
$\Delta x = x_{t+\Delta} - x_t \in \mathbb{R}^{S\times E}$, (ii) spatially
average-pools $\Delta x$ to a single global delta token in
$\mathbb{R}^{1\times E}$ and projects it to the encoder width $H$,
(iii) projects $\Delta s_t$ to $H$ and adds it to the pooled delta token
(this is the only place proprioception enters the memory pipeline),
(iv) appends $M$ learned read-out queries at the tail, and
(v) runs a small causal transformer (RMSNorm, 1D-RoPE causal
self-attention, SwiGLU FFN) so that each query attends to the fused
delta token before it. The last $M$ positions are projected to motion
dimension $D_m$ and constitute $m_t$.
Since $m_t$ requires both endpoints of window $[t,\,t\!+\!\Delta]$, it becomes
causally available only at step $t\!+\!\Delta$.

\paragraph{Memory sequence assembly.}
The final memory for an episode is the time-ordered concatenation
\begin{equation}
  \mathcal{M} \;=\; \big[\, x_{k_0}^{\mathrm{full}},\; m_{0},\; m_{\Delta},\; \dots,\; x_{k_i}^{\mathrm{TD}},\; \dots \,\big],
\end{equation}
where each entry carries its source timestamp as a positional tag.
At policy step $\tau$, the policy receives the causally valid prefix
\begin{equation}
  \mathcal{M}_\tau \;=\;
    \bigl\{\,x_{k_j}^{\cdot} : k_j \leq \tau\,\bigr\}
    \;\cup\;
    \bigl\{\,m_t : t + \Delta \leq \tau\,\bigr\},
  \label{eq:mem-causal}
\end{equation}
i.e.\ keyframe tokens whose frame index is at most $\tau$, and motion tokens
whose entire window $[t,\,t\!+\!\Delta]$ has elapsed by $\tau$.
The sequence $\mathcal{M}_\tau$ grows monotonically: at each new step the
pipeline appends newly completed tokens without recomputing earlier ones.
Keyframe selection and TokenDrop are training-free; the only learnable
parameters in the memory pipeline are those of the MotionEncoder
$f_\theta$, which is pre-trained in Stage~1 of Sec.~\ref{sec:objective}
and then frozen. The trained $f_\theta$, together with the
parameter-free keyframe and TokenDrop modules, is what ultimately writes
into $\mathcal{M}$.

\subsection{Training Objective}
\label{sec:objective}

The full system has two distinct loss surfaces, optimized in two stages:
a latent future-prediction loss $\mathcal{L}_{\mathrm{pred}}$ that trains
the MotionEncoder $f_\theta$ of Sec.~\ref{sec:memory}, and a flow-matching
loss $\mathcal{L}_{\mathrm{FM}}$ that trains the VLA backbone together
with the integration module.

\paragraph{Latent future-prediction loss for $f_\theta$.}
We train $f_\theta$ jointly with an auxiliary \emph{MotionDiT} decoder
$g_\phi$: a conditional-diffusion decoder that regresses the SigLIP
tokens of the future frame from the current tokens and the motion code.
Given $(x_t, x_{t+\Delta})$, we sample a diffusion-style timestep
$k\!\sim\!\mathcal{U}\{1,\dots,K\}$ and ask $g_\phi$ to predict the
(token-normalized) future tokens from the current tokens, the motion
code, and $k$:
\begin{equation}
  \hat x_{t+\Delta}^{(k)} \;=\; g_\phi\!\left(\, x_t,\; m_t,\; k \,\right),
  \qquad m_t = f_\theta(x_t, x_{t+\Delta}, \Delta s_t),
\end{equation}
\begin{equation}
  \mathcal{L}_{\mathrm{pred}} \;=\;
  \mathbb{E}_{t,\,k}\!\left[\,
    \big\| \hat x_{t+\Delta}^{(k)} \;-\; \mathrm{TokenNorm}(x_{t+\Delta}) \big\|_2^2
  \,\right].
\end{equation}
$\mathrm{TokenNorm}$ is a per-feature standardization applied to the
target SigLIP tokens to stabilize the regression target. $g_\phi$ is a
small DiT with AdaLN-zero timestep modulation and a tied gated
input/output projection initialized to zero, so at $k\!=\!0$ the decoder
acts as a normalized identity copy of $x_t$ and only learns residual
corrections driven by $m_t$. This zero-initialized residual head is the
sole reason the loss is stable when $f_\theta$ is randomly initialized:
gradients flow through $g_\phi$ into $f_\theta$ only via the correction
term, forcing $m_t$ to encode exactly the information needed to bridge
$x_t \to x_{t+\Delta}$. Crucially, $g_\phi$ exists only during
pre-training: at deployment the robot has already executed both endpoints
of every motion window, so memory tokens are produced by a single forward
pass through $\Phi_{\mathrm{SigLIP}}$ and $f_\theta$; no decoding is
performed and $g_\phi$ is discarded.

\paragraph{Flow-matching action loss.}
Following $\pi_{0.5}$, the policy predicts an $H$-step action chunk
$\mathbf{A}_t = [a_t, a_{t+1}, \dots, a_{t+H-1}] \in \mathbb{R}^{H\times D}$
via conditional flow matching. Given a ground-truth chunk
$\mathbf{A}_t^{\mathrm{data}}\!\sim\!p_{\mathrm{data}}$, Gaussian noise
$\mathbf{A}_t^{\mathrm{noise}}\!\sim\!\mathcal{N}(\mathbf{0},\mathbf{I})$,
and a denoising timestep $\tau\!\sim\!\mathcal{U}(0,1)$, we form the linear
interpolant
\begin{equation}
  \mathbf{A}_t^\tau \;=\; (1-\tau)\,\mathbf{A}_t^{\mathrm{data}} \;+\; \tau\,\mathbf{A}_t^{\mathrm{noise}},
  \label{eq:fm-interp}
\end{equation}
whose constant time-derivative is
$\frac{d}{d\tau}\mathbf{A}_t^\tau = \mathbf{A}_t^{\mathrm{noise}} - \mathbf{A}_t^{\mathrm{data}}$.
The action expert parameterizes a conditional velocity field
$\mathbf{v}_\theta(\mathbf{A}_t^\tau,\tau,\mathbf{h}_t)$, where the context
features $\mathbf{h}_t$ are the layer-$L$ outputs of the integrated
backbone --- so $\mathbf{h}_t$ depends on the chosen integration mechanism:
in MaC the memory enters $\mathbf{h}_t$ through the VLM expert
(\ref{eq:mac}); in MaM through the per-layer modulation
(\ref{eq:mam-attn}--\ref{eq:mam-out}); in MaE through the cross-expert
attention (\ref{eq:mae-cross}). The training objective is
\begin{equation}
  \mathcal{L}_{\mathrm{FM}} \;=\;
  \mathbb{E}_{t,\,\tau,\,\mathbf{A}_t^{\mathrm{data}},\,\mathbf{A}_t^{\mathrm{noise}}}\!\Big[\,
    \big\| \mathbf{v}_\theta\!\big(\mathbf{A}_t^\tau,\tau,\mathbf{h}_t\big)
    \;-\; \big(\mathbf{A}_t^{\mathrm{noise}} - \mathbf{A}_t^{\mathrm{data}}\big) \big\|_2^2
  \,\Big].
  \label{eq:fm-loss}
\end{equation}
At inference the action chunk is generated by integrating $\mathbf{v}_\theta$
from $\tau=1$ (pure noise) to $\tau=0$ (data) through a small number of
Euler steps.

\paragraph{Two-stage training.}
The two losses are optimized sequentially, not jointly, because the memory
pipeline encodes a task-agnostic visual-dynamics prior whereas the
integration module is task-specific.

\emph{Stage 1: Memory-pipeline pre-training.} On unlabeled robot videos we
jointly train the MotionEncoder $f_\theta$ and the auxiliary MotionDiT
decoder $g_\phi$ to minimize $\mathcal{L}_{\mathrm{pred}}$. The SigLIP and
CLIP backbones, the keyframe detector, and the TokenDrop module are
training-free throughout. After Stage~1, $g_\phi$ is discarded and
$f_\theta$ is frozen.

\emph{Stage 2: VLA fine-tuning.} For every demonstration episode we
\emph{pre-compute and cache} the full memory sequence $\mathcal{M}$
using the frozen pipeline of Stage~1.
At each training step $t$ we extract the causally valid prefix
$\mathcal{M}_t$ (Eq.~\ref{eq:mem-causal}) from the cached sequence and
project it to $\mathbf{M}_t \in \mathbb{R}^{B\times d}$; no future motion
tokens (i.e.\ windows with $t'\!+\!\Delta > t$) are visible to the policy.
SigLIP2 visual tokens are likewise pre-computed and cached, following the
$\pi_{0.5}$ recipe. Trainable parameters in
Stage~2 are exactly: (i) the VLM expert, (ii) the action expert, and (iii)
the integration module --- which contributes $0$ new parameters for MaC
(Eq.~\ref{eq:mac}), $\sim\!80$M for MaM
(Eqs.~\ref{eq:mam-attn}--\ref{eq:mam-out}), or $\sim\!190$M for MaE
(Eqs.~\ref{eq:mae-self}--\ref{eq:mae-cross}). The single optimization
objective is $\mathcal{L}_{\mathrm{FM}}$. No proprioceptive state enters
the backbone in Stage~2 (proprioception only participates in
Sec.~\ref{sec:memory} when forming motion tokens).

This decoupling is what makes the same memory pipeline reusable across all
three integration variants and across downstream tasks: Stage~1 is run
once; Stage~2 is repeated for each (task, integration) configuration.

\subsection{Implementation Details}
\label{sec:impl}

\paragraph{Memory pipeline.}
SigLIP2 provides $S\!=\!256$ patch tokens of
width $E\!=\!1152$ per $384\!\times\!384$ frame. CLIP keyframe detection
uses an RBF smoother of width $3$ frames and a threshold $\tau$ tuned so
that the average inter-keyframe gap is $\sim\!32$ frames. TokenDrop
operates on the $16\!\times\!16$ patch grid; a patch is marked significant
when its mean per-pixel absolute RGB difference w.r.t. the previous
keyframe exceeds $\delta\!=\!10^{-2}$, after which we add the patches
inside the axis-aligned bounding rectangle of the significant set. The
first keyframe is always retained in full at $S$ tokens. The MotionEncoder
uses stride $\Delta\!=\!8$, $M\!=\!8$ readout queries, an
encoder-width $H\!=\!1024$, and a $4$-layer causal transformer (RMSNorm,
1D-RoPE, SwiGLU). MotionDiT $g_\phi$ is a $6$-layer DiT with AdaLN-zero
modulation and a tied gated input/output projection initialized to zero.
All memory-pipeline parameters total $\sim\!60$M; SigLIP2 and CLIP are
frozen.

\paragraph{Memory budget and compression.}
We cap $\mathbf{M}_t$ at $B\!=\!512$ tokens of width $d\!=\!1024$ for all
integration variants, padding with zeros when the assembled $\mathcal{M}$
is shorter. Compared with the naive flattened representation of
$T\!\cdot\!S$ patch tokens per episode, this is a ${\sim}\,250\times$
reduction at $T\!\approx\!500$, while preserving both the long-range
keyframe anchors (the first keyframe contributes a full $S\!=\!256$ tokens)
and short-range dynamics ($M\!=\!8$ motion tokens per $\Delta\!=\!8$
frames).

\paragraph{Backbone and training schedule.}
The base $\pi_{0.5}$ policy processes $576$ context tokens ($512$ visual
$+$ $64$ language) and contains $3.2$B parameters. Stage~2 fine-tuning uses
AdamW ($\beta_1\!=\!0.9$, $\beta_2\!=\!0.95$, weight decay $0$), constant
learning rate $5\!\times\!10^{-5}$ with $5\%$ warmup, gradient-norm clip
$1.0$, EMA $0.999$, action horizon $H\!=\!20$, and $80$k steps. Effective
batch size is $64$ for MaC and MaM and $8$ for MaE (the larger memory
expert is the bottleneck). All runs use $4{\times}$NVIDIA~A40 GPUs with
FSDP, $\sim\!3$--$4$ days per configuration. Stage~1 uses the same
optimizer with a shorter $40$k-step schedule on a separate unlabeled
video corpus. The SigLIP2 ViT is frozen and its outputs are cached on
disk to amortize cost across configurations.
