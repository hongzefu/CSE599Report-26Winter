\begin{abstract}
Long-horizon robotic manipulation is non-Markovian: counting events,
recovering object identity under occlusion, and reproducing demonstrated
motions all require retaining many seconds of visual history while the
policy runs at control rate within a fixed token budget. Recent benchmark
evidence~\citep{dai2026robomme} shows that perceptual memory representations
dominate motion-centric and time-sensitive tasks, but its concrete
instantiations --- uniform frame sampling and temporal-saliency-agnostic
token dropping --- are content-blind and degrade sharply as the memory
budget tightens, while symbolic / language-subgoal memory cannot adequately
represent low-level continuous motion. We introduce a robot-specialized
perceptual memory that decouples \emph{what} the robot saw from \emph{how}
things moved between observations: a CLIP-derivative keyframe selector with
delta-based per-keyframe TokenDrop carries sparse spatial evidence at
semantic transitions, and a learned MotionEncoder, pretrained on unlabeled
robot video by JEPA-style~\citep{assran2023ijepa} latent future-token
prediction, summarizes each short window into a single compact motion
token. The combined sequence is exposed to the $\pi_{0.5}$ backbone
through a fixed-budget memory interface compatible with all three
Memory-as-Context, -Modulator, and -Expert integration mechanisms used in
the MME-VLA suite~\citep{dai2026robomme}, enabling apples-to-apples
comparison against uniform-frame-sampling and token-dropping baselines at
matched memory budgets. Full quantitative evaluation on RoboMME is
\emph{in progress} (Sec.~\ref{sec:exp}).
\end{abstract}


\section{Introduction}
\label{sec:intro}

Open-world robotic manipulation is rarely Markovian. A household robot that
returns books to the right shelves must remember where each book originally
came from; a robot asked to ``press the button \emph{exactly} when the
moving cube reaches the target a specified number of times'' must
remember an event count and an event ordering; a robot reproducing a
demonstrated trajectory must replay motion patterns it observed seconds
earlier~\citep{dai2026robomme}. Modern vision-language-action (VLA) policies
run at control rate, on a transformer with a fixed input window, so the
central design question for memory-augmented manipulation is not
\emph{whether} to retain history but \emph{which projection of it} fits
inside that window without losing decision-relevant signal.

Existing memory designs fall into two families, each with limitations.
\emph{Symbolic} memory~\citep{bu2025univla,sridhar2025memer,geminiroboticsTeam2025gemini15}
discretizes history into language subgoals; controlled evaluation~\citep{dai2026robomme}
shows it under-specifies continuous motion. \emph{Perceptual} memory retains
raw visual tokens~\citep{jang2025contextvla,koo2025hamlet,fang2025sam2act,shi2026memoryvla,physicalintelligence2026mem}
and is empirically stronger, but the two perceptual instantiations in that study
--- uniform frame sampling and saliency-agnostic token dropping --- are
content-blind and degrade steeply as the memory budget shrinks.

Robot videos have complementary structure: a few moments that genuinely
matter (contact events, viewpoint shifts) and predictable motion in between.
The right memory should mirror that: semantic keyframes for \emph{what was
seen and when}, and a compact learned summary for \emph{how things moved}.
We train a MotionEncoder JEPA-style~\citep{assran2023ijepa} to predict
future SigLIP latents, forcing each motion token to encode exactly what
content-blind subsampling discards.

Concretely: (1)~a CLIP-derivative keyframe detector plus delta-based
TokenDrop produces sparse but semantically anchored keyframe patches;
(2)~a MotionEncoder $f_\theta$ compresses each short window into one motion
token (decoder discarded after pre-training); (3)~the fixed-budget memory
$\mathbf{M}_t$ plugs into the $\pi_{0.5}$ backbone via the same MaC / MaM /
MaE interfaces as MME-VLA~\citep{dai2026robomme}, enabling apples-to-apples
comparison against content-blind baselines at matched budgets.

\paragraph{Contributions.}
\begin{itemize}
    \item A perceptual memory that couples CLIP-scored keyframes with
          delta-based TokenDrop and a learned MotionEncoder, decoupling
          spatial evidence from motion summaries in a fixed token budget.
    \item A JEPA-style latent prediction objective~\citep{assran2023ijepa}
          with an auxiliary MotionDiT decoder (discarded at deployment)
          that forces each motion token to encode the information needed
          to bridge consecutive observations.
    \item A controlled evaluation on RoboMME~\citep{dai2026robomme}:
          our memory plugs into the MaC / MaM / MaE interfaces of
          MME-VLA on $\pi_{0.5}$, enabling matched-budget comparison
          against content-blind baselines (\emph{evaluation in progress}).
\end{itemize}


\section{Related Work}
\label{sec:related}

\paragraph{Memory-based Robotic Manipulation Models.}
History-dependent manipulation has motivated a diverse family of memory
designs. Context-based approaches concatenate past visual tokens
(ContextVLA~\citep{jang2025contextvla}) or compress them recurrently
(BC-RNN~\citep{mandlekar2021bcrnn}; Mamba-style: RoboMamba~\citep{liu2024robomamba},
MITL~\citep{mitl2025}), but either pay linear attention cost or compress
without selection. Explicit memory-bank methods retrieve on demand:
Hamlet~\citep{koo2025hamlet}, MemoryVLA~\citep{shi2026memoryvla}, and
SAM2Act~\citep{fang2025sam2act} cache visual or segmentation features;
MEM~\citep{physicalintelligence2026mem} organizes them at multiple temporal
resolutions; BPP~\citep{sobol2026bpp} explicitly selects key history frames.
A third direction offloads memory to a VLM (UniVLA~\citep{bu2025univla},
MemER~\citep{sridhar2025memer}, Gemini-Robotics-1.5~\citep{geminiroboticsTeam2025gemini15}),
at the cost of modular inference and broken end-to-end optimization. These
works differ in backbone, stored representation, and injection mechanism,
making head-to-head comparison difficult --- the gap our study targets.

\paragraph{Efficient Video Representation for Robotic Manipulation.}
Robot videos are heavily redundant, yet policies must retain seconds of
history at control rate within a fixed token window. Prior work aggregates
at the encoder side (ContextVLA~\citep{jang2025contextvla},
Hamlet~\citep{koo2025hamlet}, long-context diffusion~\citep{torne2025longcontext})
or selects key frames (BPP~\citep{sobol2026bpp}, SAM2Act~\citep{fang2025sam2act}).
General video methods (TimeChat-Online~\citep{yao2025timechat},
VideoLLM-online~\citep{chen2024videollmonline}) show that ${\sim}80\%$ of
tokens are redundant, but manipulation demands tighter budgets still. We
combine delta-based TokenDrop over semantic keyframes with a MotionEncoder
summary, decoupling sparse spatial evidence from short-range dynamics within
a single fixed-budget representation.

\subsection{Model Architecture}
\label{sec:arch}

Our policy is a memory-augmented vision-language-action (VLA) model built on
top of the $\pi_{0.5}$ backbone. We initialize from the pre-trained backbone
and fine-tune its experts end-to-end in Stage~2 (Sec.~\ref{sec:objective}).
The only components kept frozen throughout both stages are the SigLIP2 visual
encoder and, after Stage~1, the MotionEncoder $f_\theta$.
We inject the per-episode memory sequence assembled in
Sec.~\ref{sec:memory} as additional context tokens.

\paragraph{$\pi_{0.5}$ backbone.}
At history timestep $t$, the policy receives language tokens
$\boldsymbol{\ell} \in \mathbb{R}^{N_\ell\times d}$ and multi-view image
tokens $\mathbf{o}_t \in \mathbb{R}^{N_o\times d}$ extracted by a frozen
SigLIP2 encoder. Following the $\pi_0$/$\pi_{0.5}$ multi-expert
design~\citep{black2024pi0,black2025pi05}, the backbone is composed of a
\emph{VLM expert} $E_{\mathrm{vlm}}$ that fuses language and vision and an
\emph{action expert} $E_{\mathrm{act}}$ that predicts an action chunk
conditioned on the VLM features and a denoising timestep $\tau$. Both
experts are $L$-layer transformers connected through blockwise causal
attention; the action expert is further conditioned on $\tau$ via
AdaLN-Zero modulation~\citep{peebles2023scalable}. We refer the reader to
the original papers for the exact equations. Unlike the original
$\pi_{0.5}$, which uses the VLM expert to also generate low-level language
commands, we keep the VLM as a pure feature extractor; this leaves a clean
interface at which memory can be injected.

\paragraph{Memory token block.}
To enable history-dependent reasoning, we augment the policy with an explicit
memory state $\mathbf{M}_t$:
\begin{equation}
  a_t \;\sim\; \pi\!\left(\cdot \;\middle|\; l,\; \mathbf{o}_t,\; \mathbf{M}_t\right),
  \label{eq:mem-policy}
\end{equation}
where $\mathbf{M}_t$ is the length-$B$ projection of the causal prefix
$\mathcal{M}_t$ (Eq.~\ref{eq:mem-causal}), containing only observations and
motion windows that have fully elapsed before step $t$.
For fair comparison across designs, we allocate a fixed memory token budget $B$
for all memory-based models: each representation produces at most $B$ tokens;
if fewer are available, the sequence is zero-padded.
Thus all memory variants share the same interface
\begin{equation}
  \mathbf{M}_t \;\in\; \mathbb{R}^{B\times d}.
  \label{eq:mem-interface}
\end{equation}
We expose the memory sequence $\mathcal{M}$ produced in
Sec.~\ref{sec:memory} to the backbone as this token block.
The two-token-kind structure of $\mathcal{M}$ — TokenDropped keyframe patches
and MotionEncoder motion tokens — is opaque to the backbone: from the policy's
perspective $\mathbf{M}_t$ is just a length-$B$ sequence of $d$-dimensional
vectors carrying time tags.

\paragraph{Three integration mechanisms.}
We systematically study three places at which $\mathbf{M}_t$ can enter the
backbone, trading parameter count, attention budget, and isolation of the
pretrained VLM features against each other.

\emph{(1) Memory-as-Context (MaC).} Memory tokens are concatenated with the
standard image and language inputs of the VLM expert and jointly processed.
This introduces no new parameters but lets memory compete with raw
observation tokens for the same attention budget inside $E_{\mathrm{vlm}}$.

\emph{(2) Memory-as-Modulator (MaM).} The action latent at each layer
cross-attends $\mathbf{M}_t$, and the result is mapped to per-feature
scale/shift parameters through a zero-initialized MLP composed with
AdaLN. The modulation is initialized to the identity, so fine-tuning starts
from the pretrained $\pi_{0.5}$ behavior. MaM adds $\sim\!80$M parameters
and leaves the VLM expert untouched.

\emph{(3) Memory-as-Expert (MaE).} A dedicated third expert
$E_{\mathrm{mem}}$ processes $\mathbf{M}_t$ through its own $L$ transformer
layers. Under an asymmetric blockwise causal mask, the VLM expert and
memory expert use self-attention only and do not attend each other, while
the action expert is the sole consumer that attends both as additional
context. This preserves the pretrained $\pi_{0.5}$ representations exactly
and allocates a separate transformer's worth of capacity ($\sim\!190$M
parameters) to memory processing.

\subsection{Memory Token Generation and Training}
\label{sec:memory}
% Scope: this subsection describes how the per-episode memory tokens consumed
% by the downstream VLA are produced and trained. The downstream injection
% mechanism (how the VLA attends to or interleaves these tokens) is described
% in Sec.~\ref{sec:arch}.

As the robot executes, the memory $\mathcal{M}_\tau$ grows online from the
history of observations accumulated prior to policy step $\tau$.  It is a
sparse, time-aligned sequence of two complementary token kinds:
(i) \emph{keyframe tokens}, which anchor the scene at semantic transitions,
and (ii) \emph{motion tokens}, fixed-stride summaries of short-range
visual change produced by a learned MotionEncoder. Both kinds are encoded
strictly from already-elapsed frames: keyframe tokens are committed as soon
as a keyframe is detected, and a motion token $m_t$ over window
$[t,\,t\!+\!\Delta]$ becomes available only after step $t\!+\!\Delta$ has
passed (Eq.~\ref{eq:mem-causal}). All encoding is performed by frozen
perception backbones independently of the VLA; no gradient flows back into
these encoders during VLA training.

\paragraph{Keyframe detection.}
Let $\{I_t\}_{t \geq 0}\!\subset\!\mathbb{R}^{3\times H\times W}$ denote the sequence of RGB frames observed during execution.
Keyframes mark moments of meaningful scene change (object pickup,
contact events, viewpoint shifts). Each frame is scored by the smoothed
discrete derivative of a frozen CLIP encoder $\Phi_{\mathrm{CLIP}}$:
\begin{equation}
  d_t \;=\; \big\| \Phi_{\mathrm{CLIP}}(I_t) - \Phi_{\mathrm{CLIP}}(I_{t-1}) \big\|_2,
  \qquad
  \tilde d_t \;=\; (k_{\mathrm{rbf}} * d)_t,
\end{equation}
where $k_{\mathrm{rbf}}$ is a small RBF kernel that suppresses frame-rate
jitter. Keyframes are local maxima of $\tilde d_t$ exceeding a threshold
$\tau$, with the first frame $t=0$ always retained as an anchor. The
detector is purely deterministic preprocessing and has no learnable
parameters; $\tau$ is a single hyperparameter chosen so that the average
inter-keyframe gap is $\sim\!32$ frames.

\paragraph{Keyframe TokenDrop.}
Storing the full $S$-token grid for every keyframe is wasteful: most of
the image (background, robot base, table) repeats across keyframes. For
every keyframe $k_i$ with $i\!>\!0$ we retain only the patches that
actually changed relative to its \emph{previous} keyframe $k_{i-1}$.
Concretely, on the $16\!\times\!16$ patch grid we mark a patch as
\emph{significant} if its mean per-pixel absolute difference between
$I_{k_i}$ and $I_{k_{i-1}}$ exceeds $\delta$, and we keep the union of
all significant patches together with the patches lying inside their
axis-aligned bounding rectangle (the latter preserves spatial contiguity
for downstream attention). The first keyframe $k_0$ is always kept in
full ($S$ tokens); each subsequent keyframe contributes
$|\mathcal{P}_i|\!\le\!S$ tokens. TokenDrop is also parameter-free.

\paragraph{Motion tokens.}
Let $\Phi_{\mathrm{SigLIP}}\!:\!\mathbb{R}^{3\times H\times W}\!\to\!\mathbb{R}^{S\times E}$ ($S\!=\!256$ patches, $E\!=\!1152$) be the frozen SigLIP tokenizer, and let $x_t \!=\! \Phi_{\mathrm{SigLIP}}(I_t)\in\mathbb{R}^{S\times E}$; $\{s_t\}$ denotes the proprioceptive state sequence.
Independently of keyframe placement, we tile the video with non-overlapping
windows of fixed stride $\Delta$ (we use $\Delta\!=\!8$). For each window
$[t,\, t\!+\!\Delta]$ a trainable \emph{MotionEncoder}
$f_\theta$ compresses the visual change into $M$ latent tokens
$m_{t}\!\in\!\mathbb{R}^{M\times D_m}$:
\begin{equation}
  m_t \;=\; f_\theta\!\left(\, x_t,\; x_{t+\Delta},\; \Delta s_t \,\right),
  \qquad \Delta s_t = s_{t+\Delta} - s_t.
\end{equation}
Internally, $f_\theta$ (i) computes the patchwise visual delta
$\Delta x = x_{t+\Delta} - x_t \in \mathbb{R}^{S\times E}$, (ii) spatially
average-pools $\Delta x$ to a single global delta token in
$\mathbb{R}^{1\times E}$ and projects it to the encoder width $H$,
(iii) projects $\Delta s_t$ to $H$ and adds it to the pooled delta token
(this is the only place proprioception enters the memory pipeline),
(iv) appends $M$ learned read-out queries at the tail, and
(v) runs a small causal transformer (RMSNorm, 1D-RoPE causal
self-attention, SwiGLU FFN) so that each query attends to the fused
delta token before it. The last $M$ positions are projected to motion
dimension $D_m$ and constitute $m_t$.
Since $m_t$ requires both endpoints of window $[t,\,t\!+\!\Delta]$, it becomes
causally available only at step $t\!+\!\Delta$.

\paragraph{Memory sequence assembly.}
The final memory for an episode is the time-ordered concatenation
\begin{equation}
  \mathcal{M} \;=\; \big[\, x_{k_0}^{\mathrm{full}},\; m_{0},\; m_{\Delta},\; \dots,\; x_{k_i}^{\mathrm{TD}},\; \dots \,\big],
\end{equation}
where each entry carries its source timestamp as a positional tag.
At policy step $\tau$, the policy receives the causally valid prefix
\begin{equation}
  \mathcal{M}_\tau \;=\;
    \bigl\{\,x_{k_j}^{\cdot} : k_j \leq \tau\,\bigr\}
    \;\cup\;
    \bigl\{\,m_t : t + \Delta \leq \tau\,\bigr\},
  \label{eq:mem-causal}
\end{equation}
i.e.\ keyframe tokens whose frame index is at most $\tau$, and motion tokens
whose entire window $[t,\,t\!+\!\Delta]$ has elapsed by $\tau$.
The sequence $\mathcal{M}_\tau$ grows monotonically: at each new step the
pipeline appends newly completed tokens without recomputing earlier ones.
Keyframe selection and TokenDrop are training-free; the only learnable
parameters in the memory pipeline are those of the MotionEncoder
$f_\theta$, which is pre-trained in Stage~1 of Sec.~\ref{sec:objective}
and then frozen. The trained $f_\theta$, together with the
parameter-free keyframe and TokenDrop modules, is what ultimately writes
into $\mathcal{M}$.

\subsection{Training Objective}
\label{sec:objective}

The full system has two distinct loss surfaces, optimized in two stages:
a latent future-prediction loss $\mathcal{L}_{\mathrm{pred}}$ that trains
the MotionEncoder $f_\theta$ of Sec.~\ref{sec:memory}, and a flow-matching
loss $\mathcal{L}_{\mathrm{FM}}$ that trains the VLA backbone together
with the integration module.

\paragraph{Latent future-prediction loss for $f_\theta$.}
We train $f_\theta$ jointly with an auxiliary \emph{MotionDiT} decoder
$g_\phi$ (Fig.~\ref{fig:training}): a conditional-diffusion decoder that regresses the SigLIP
tokens of the future frame from the current tokens and the motion code.
Given $(x_t, x_{t+\Delta})$, we sample a diffusion-style timestep
$k\!\sim\!\mathcal{U}\{1,\dots,K\}$ and ask $g_\phi$ to predict the
(token-normalized) future tokens from the current tokens, the motion
code, and $k$:
\begin{equation}
  \hat x_{t+\Delta}^{(k)} \;=\; g_\phi\!\left(\, x_t,\; m_t,\; k \,\right),
  \qquad m_t = f_\theta(x_t, x_{t+\Delta}, \Delta s_t),
\end{equation}
\begin{equation}
  \mathcal{L}_{\mathrm{pred}} \;=\;
  \mathbb{E}_{t,\,k}\!\left[\,
    \big\| \hat x_{t+\Delta}^{(k)} \;-\; \mathrm{TokenNorm}(x_{t+\Delta}) \big\|_2^2
  \,\right].
\end{equation}
$\mathrm{TokenNorm}$ is a per-feature standardization applied to the
target SigLIP tokens to stabilize the regression target. $g_\phi$ is a
small DiT with AdaLN-zero timestep modulation and a tied gated
input/output projection initialized to zero, so at $k\!=\!0$ the decoder
acts as a normalized identity copy of $x_t$ and only learns residual
corrections driven by $m_t$. This zero-initialized residual head is the
sole reason the loss is stable when $f_\theta$ is randomly initialized:
gradients flow through $g_\phi$ into $f_\theta$ only via the correction
term, forcing $m_t$ to encode exactly the information needed to bridge
$x_t \to x_{t+\Delta}$. Crucially, $g_\phi$ exists only during
pre-training: at deployment the robot has already executed both endpoints
of every motion window, so memory tokens are produced by a single forward
pass through $\Phi_{\mathrm{SigLIP}}$ and $f_\theta$; no decoding is
performed and $g_\phi$ is discarded.

\paragraph{Flow-matching action loss.}
Following $\pi_{0.5}$, the policy predicts an $H$-step action chunk
$\mathbf{A}_t \in \mathbb{R}^{H\times D}$ via conditional flow matching:
the action expert parameterizes a velocity field
$\mathbf{v}_\theta(\mathbf{A}_t^\tau,\tau,\mathbf{h}_t)$ trained to match
the noise-to-data transport direction,
\begin{equation}
  \mathcal{L}_{\mathrm{FM}} \;=\;
  \mathbb{E}\!\left[\,
    \big\| \mathbf{v}_\theta\!\big(\mathbf{A}_t^\tau,\tau,\mathbf{h}_t\big)
    \;-\; \big(\mathbf{A}_t^{\mathrm{noise}} - \mathbf{A}_t^{\mathrm{data}}\big) \big\|_2^2
  \,\right],
  \label{eq:fm-loss}
\end{equation}
where the context features $\mathbf{h}_t$ are the layer-$L$ outputs of the
integrated backbone and therefore depend on the chosen integration
mechanism (MaC, MaM, or MaE). At inference the action chunk is generated
by integrating $\mathbf{v}_\theta$ from $\tau\!=\!1$ to $\tau\!=\!0$ through
a small number of Euler steps.

\paragraph{Two-stage training.}
The two losses are optimized sequentially, not jointly, because the memory
pipeline encodes a task-agnostic visual-dynamics prior whereas the
integration module is task-specific.

\emph{Stage 1: Memory-pipeline pre-training.} On unlabeled robot videos we
jointly train the MotionEncoder $f_\theta$ and the auxiliary MotionDiT
decoder $g_\phi$ to minimize $\mathcal{L}_{\mathrm{pred}}$. The SigLIP and
CLIP backbones, the keyframe detector, and the TokenDrop module are
training-free throughout. After Stage~1, $g_\phi$ is discarded and
$f_\theta$ is frozen.

\emph{Stage 2: VLA fine-tuning.} For every demonstration episode we
pre-compute and cache the full memory sequence $\mathcal{M}$ using the
frozen pipeline of Stage~1, together with SigLIP2 visual tokens. At each
training step $t$ the causally valid prefix
$\mathcal{M}_t$ (Eq.~\ref{eq:mem-causal}) is projected to
$\mathbf{M}_t \in \mathbb{R}^{B\times d}$ and fed to the chosen integration
module; the VLM expert, action expert, and integration module are trained
end-to-end on $\mathcal{L}_{\mathrm{FM}}$. No proprioceptive state enters
the backbone in Stage~2 --- proprioception only participates in
Sec.~\ref{sec:memory} when forming motion tokens.

This decoupling is what makes the same memory pipeline reusable across all
three integration variants and across downstream tasks: Stage~1 is run
once; Stage~2 is repeated for each (task, integration) configuration.

\subsection{Implementation Details}
\label{sec:impl}

% \paragraph{Memory pipeline.}
% SigLIP2 provides $S\!=\!256$ patch tokens of
% width $E\!=\!1152$ per $256\!\times\!256$ frame. CLIP keyframe detection
% uses an RBF smoother of width $3$ frames and a threshold $\tau$ tuned so
% that the average inter-keyframe gap is $\sim\!32$ frames. TokenDrop
% operates on the $16\!\times\!16$ patch grid; a patch is marked significant
% when its mean per-pixel absolute RGB difference w.r.t. the previous
% keyframe exceeds $\delta\!=\!10^{-2}$, after which we add the patches
% inside the axis-aligned bounding rectangle of the significant set. The
% first keyframe is always retained in full at $S$ tokens. The MotionEncoder
% uses an $8$-frame window at stride $\Delta\!=\!16$, $M\!=\!1$ readout query, an
% encoder-width $H\!=\!1024$, and a $4$-layer causal transformer (RMSNorm,
% 1D-RoPE, SwiGLU). MotionDiT $g_\phi$ is a $6$-layer DiT with AdaLN-zero
% modulation and a tied gated input/output projection initialized to zero.
% All memory-pipeline parameters total $\sim\!60$M; SigLIP2 and CLIP are
% frozen.

\paragraph{Memory budget and compression.}
We cap the keyframe-token memory at $B\!=\!2000$ tokens of width
$d\!=\!1024$; the MotionEncoder contributes up to $125$ motion tokens
separately, giving a combined budget of $2000{+}125$, padding with zeros
when shorter.  Compared with the naive flattened representation of
$T\!\cdot\!S$ patch tokens per episode, this is a ${\sim}\,250\times$
reduction at $T\!\approx\!2000$, while preserving both the long-range
keyframe anchors (the first keyframe contributes a full $S\!=\!256$
tokens) and short-range dynamics ($M\!=\!1$ motion token per
$\Delta\!=\!16$-frame window).

\section{Experiments}
\label{sec:exp}

Our evaluation is structured around three research questions.
\textbf{RQ1 (memory content).} Under a fixed token budget and a fixed
integration mechanism, do content-aware perceptual memories (CLIP keyframes,
TokenDrop, learned motion tokens) preserve more decision-relevant signal
than the content-blind subsampling baselines used in
MME-VLA~\citep{dai2026robomme}?
\textbf{RQ2 (budget sensitivity).} How does each memory variant degrade as
the memory budget $B$ shrinks --- gracefully, or with a cliff?
\textbf{RQ3 (integration).} Among Memory-as-Context, Memory-as-Modulation,
and Memory-as-Expert, which integration mechanism best exploits a given
memory representation?
At the time of submission, the budget-sensitivity component of RQ2 has
finished for both content-blind baselines, while the keyframe-based
configurations and the cross-backbone study addressing RQ1 and RQ3 are
still running; we report what is currently available and explicitly mark
all unfinished cells as \emph{ongoing} (Sec.~\ref{sec:main-results}).

\subsection{Experimental Setup}
\label{sec:setup}

\paragraph{Benchmark.}
We evaluate on RoboMME~\citep{dai2026robomme}, a long-horizon,
history-dependent manipulation suite spanning four memory categories
(temporal, spatial, object, procedural). It provides matched train/eval
splits, a unified scoring protocol, and the MME-VLA reference policies,
making it the natural testbed for controlled memory-content comparisons.
Compared methods are detailed in Sec.~\ref{sec:main-results}.

\paragraph{Metric.}
Following RoboMME~\citep{dai2026robomme}, we report task Success Rate
(\%) averaged over the benchmark's evaluation episodes; higher is better.

\paragraph{Training and inference.}
We follow the two-stage recipe of Sec.~\ref{sec:objective}: Stage~1
pre-trains the MotionEncoder $f_\theta$ once on unlabeled robot video with
$\mathcal{L}_{\mathrm{pred}}$, after which $f_\theta$ is frozen and reused
across every (memory, integration, backbone) configuration; Stage~2
fine-tunes the policy with $\mathcal{L}_{\mathrm{FM}}$ for $80$k steps
under the optimizer and hardware setup described in Sec.~\ref{sec:impl}.
At inference the memory pipeline runs strictly causally
(Eq.~\ref{eq:mem-causal}): only motion windows whose right endpoint has
elapsed before the current step contribute to $\mathbf{M}_t$.

\subsection{Main Results}
\label{sec:main-results}

This subsection collects the results currently available against the
memory-content question (RQ1) and the budget-sensitivity question (RQ2).
The completed runs cover the two content-blind perceptual baselines
(\emph{Even Frame Sampling} and \emph{Even TokenDropping}) across the
full budget sweep $B\!\in\!\{1024,512,256,128,64\}$ under MaM; the
keyframe-based configurations and the cross-backbone study are still
running and appear as \emph{ongoing} entries in Tab.~\ref{tab:main}.

\paragraph{Baselines.}
All configurations share the $\pi_{0.5}$ backbone and MaM integration,
so differences are attributable to memory \emph{content}:
\begin{itemize}
  \item \textbf{MME-VLA Clip-Keyframe}: retains every token of selected
        keyframes without compression (${\sim}8000{+}$ tokens; upper bound).
  \item \textbf{MME-VLA Clip-Keyframe + TokenDrop}: adds delta-based patch
        retention, compressing to a fixed budget of $2000$ tokens.
  \item \textbf{MME-VLA Clip-KF + TokenDrop + Motion (Ours)}: further
        appends $125$ motion tokens, yielding $2000{+}125$ total.
  \item \textbf{MME-VLA Even Frame Sampling (EFS)}: uniformly spaced
        frames with pooled tokens, swept over $B\!\in\!\{1024,512,256,128,64\}$.
  \item \textbf{MME-VLA Even TokenDropping (ETD)}: saliency-agnostic patch
        dropping, swept over the same five budgets.
  \item \textbf{Alternative backbones} (OpenVLA, Qwen2.5-VL): same memory
        pipeline on different VLA families, probing transferability.
\end{itemize}

\paragraph{Findings on the content-blind baselines.}
Two observations follow from Tab.~\ref{tab:main}.
\emph{(i)} EFS outperforms ETD at every budget ($+5.7$ pt at $B\!=\!1024$,
widening to $+12.3$ pt at $B\!=\!64$): full frames carry more usable signal
than the same number of arbitrarily selected patches.
\emph{(ii)} EFS degrades gradually ($15.45$ pt over the full sweep),
while ETD collapses sharply between $B\!=\!128$ and $B\!=\!64$
($14.65$ pt single-step drop), suggesting content-blind patch removal
crosses a representational threshold at tight budgets.

\subsection{Analysis}
\label{sec:analysis}

These findings are consistent with our motivation in Sec.~\ref{sec:intro}:
content-blind patch removal crosses a representational threshold at tight
budgets, while frame-level subsampling degrades more gracefully by always
delivering a complete spatial layout. The analysis is restricted to the
two completed sweeps; per-task breakdowns and failure cases will be
reported once the \textit{ongoing} runs finish.


\section{Conclusion}
\label{sec:conclusion}

We proposed a robot-specialized perceptual memory (Sec.~\ref{sec:method})
that decouples semantic keyframes from learned short-range motion
summaries within a fixed token budget, exposed to the $\pi_{0.5}$
backbone through three interchangeable interfaces (MaC / MaM / MaE).
The completed budget sweeps establish that the two content-blind
perceptual variants of MME-VLA degrade in qualitatively different ways:
\emph{Even Frame Sampling} loses $15.45$ pt nearly linearly across
$B\!\in\![1024,64]$, while \emph{Even TokenDropping} loses $22.06$ pt
with a sharp $14.65$-pt cliff at $B\!=\!64$, confirming that which
patches are retained matters most when the budget is tightest --- the
regime our content-aware design targets.

\paragraph{Limitations and future work.}
Quantitative evaluation is incomplete at submission; the keyframe-based
configurations, integration-mechanism comparison (RQ3), and cross-backbone
study are still running, with full results to appear in the camera-ready
version. Future directions include (i)~online keyframe reassignment as
episodes evolve; (ii)~hybrid perceptual-plus-language memories combining
our tokens with VLM-driven subgoal representations; and (iii)~validation
on real hardware where sensor noise will further stress the keyframe
detector and MotionEncoder.
